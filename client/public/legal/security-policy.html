Security Policy for CVitaePRO

Effective Date: March 26, 2025

At One Guy Productions, LLC, we take the security of user data seriously. This Security Policy outlines the technical and organizational measures we use to protect personal data and maintain the integrity of the CVitaePRO platform.

1. Data Encryption

  • In Transit: All data is encrypted using TLS 1.2 or higher.
  • At Rest: Sensitive data (e.g., user credentials) is encrypted or hashed using industry standards such as bcrypt.

2. Authentication & Access Control

  • Passwords are hashed using bcrypt and never stored in plain text.
  • User roles and authentication tokens are validated on every request.
  • Admin and developer access to infrastructure is restricted via role-based access and protected with 2FA.

3. Infrastructure Security

  • All cloud infrastructure is hosted via industry-compliant providers (e.g., Render, AWS).
  • Firewalls and network-level security are enabled by default.
  • Database access is IP-restricted and encrypted.

4. Monitoring and Logging

  • Logs are maintained for authentication events, API usage, and error events.
  • Suspicious activity (e.g., excessive requests, brute-force login attempts) is flagged and rate-limited.

5. Backups and Redundancy

  • Regular backups of critical data are performed and stored securely.
  • Backup data is encrypted and retained in accordance with our data retention policy.

6. Security Testing

  • CVitaePRO undergoes internal code reviews and automated vulnerability scans.
  • Patches and updates are deployed regularly to address known CVEs and zero-day vulnerabilities.

7. Incident Response

In the event of a data breach:

  • Affected users will be notified within 72 hours (per GDPR Article 33).
  • A full investigation and mitigation report will be conducted and made available.

8. Your Responsibilities

Users are responsible for:

  • Using strong passwords and updating them periodically
  • Not sharing login credentials with third parties
